\subsection{Exploiting the vulnerability}

Being able to specify the string format specifier allows as to use \emph{printf} method in order to either crash a program, look up values on stack and memory and even modify content at specific memory locations with a specified value. We used the following features of \emph{printf} specifier string:
\begin{itemize}
	\item \%x - prints hexadecimal value provided in the arguments list
	\item \%s - prints value at the current memory location from the heap provided by arugments list
	\item \%w - writes number of bytes written so far to the output at the current location provided by arguments list
\end{itemize}
It's important to realize that argument list for printf would be kept on the stack. Therefore we can use \emph{printf} to traverse the stack using for instance many \%x in the input string.

\subsection{Crashing the program}
In order to crash the program it was enough to provide a string as input that contained a few "\%s". This way values popped from the stack would be provided to the \emph{printf} method and memory address at the locations specified by the values. At some point we would try access memory that the process couldn't access which results in a segmentation fault and a crash of the program.

Hard links are used all over the operating system to point to given location in file-system. Therefore regardless of jail the link always points to the same location in file-system.
